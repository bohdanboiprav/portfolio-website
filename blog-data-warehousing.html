DATA WAREHOUSING

  • by:Bohdan Boiprav
  • date:Mar 9, 2023

Data Warehousing refers to the process of collecting data from different relevant sources and storing it into a central repository after making necessary transformations to make the data suitable for analytics.

Having only a database that is accessed by both users and analysts will cause the problems of slow data retrieval as both parties would be competing for access and it would be difficult to keep track of changes.

Data warehouse solutions

- Azure Synapse Analytics
- Amazon Redshift
- Google BigQuery

Features of a Data Warehouse

1. Historical data storage
If you have a database that stores only current data, then do observe the trends of change in data according to time would be difficult and hence we need to store historical data in a data warehouse where we can use to make data driven business decisions.

2. Integration
You can integrate data from multiple sources and formats into a single version of truth enabling the data to be less ambigious and have a well structured schema.

3. Data Quality
The data warehouse stores the data after applying certain transformation which involve data cleaning, validation and normalization to make it compatible and easy to access by the analytics team.

4. Data Security
The access to data warehouse is only to the people who are authorized and there are several encryption and auditing mechanisms to identify any fraudualent activity.

There are two very famous data warehouse architectures

Inmon data warehouse architecture

Bill Inmon, the father of data warehousing believed in a unified source of information which is non-redundant, clean, structured and hence had to be stored in 3NF normalization format.

Inmonâ€™s method is a top-down approach, where the data warehouse is broken down into department data marts representing different systems in a company be it product, sales, HR and finance and the data is divided according to their specific needs.

The central data warehouse is called the Enterprise Data Warehouse. And this ensures the data integrity and consistency accross all systems.

Kimball data warehouse architecture

This architecture was proposed by Ralph Kimball in which he focused on making data marts with dimensional data modeling according to specific business requirements and then once all the data had been loaded into data marts, it would combined inside the the data warehouse.

It is a bottom approach where the key features of the data are first analysed according to the critical business processes and questions followed by relevant ETL and storage into star or snowflake schemas.
